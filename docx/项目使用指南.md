# 项目使用指南

## 1. 项目用途

这个项目用于完成视网膜血管分割任务，主要包含四部分功能：

- 数据准备
- 模型训练
- 模型预测
- Web 页面演示

如果你只想把项目跑通，按下面步骤操作即可。

## 2. 第一步：安装环境

建议先进入项目目录，再创建虚拟环境：

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

## 3. 第二步：准备 DRIVE 数据集

将数据集解压到以下目录：

```text
data/DRIVE/
```

确认结构类似下面这样：

```text
data/DRIVE/
├── training/
│   ├── images/
│   ├── 1st_manual/
│   └── mask/
└── test/
    ├── images/
    └── mask/
```

## 4. 第三步：划分数据

运行：

```bash
python main.py --mode prepare-data --config configs/base.yaml
```

运行完成后会生成：

```text
data/train/images
data/train/masks
data/val/images
data/val/masks
data/test/images
```

说明：

- `train` 用于训练
- `val` 用于验证
- `test` 用于预测展示

## 5. 第四步：开始训练

运行：

```bash
python main.py --mode train --config configs/base.yaml
```

训练时会在终端输出每个 epoch 的指标，例如：

- `train_loss`
- `train_dice`
- `val_loss`
- `val_dice`

模型文件会保存在：

```text
experiments/
```

重点关注：

- `experiments/best.pt`
- `experiments/last.pt`

## 6. 第五步：什么时候停训练

建议主要看验证集指标 `val_dice`。

可以按下面方式判断：

- 如果 `val_dice` 持续上升，可以继续训练
- 如果连续很多轮几乎不再提升，可以停止训练
- 最终优先使用 `best.pt`

停止训练不会让模型丢失，只要权重文件已经保存下来，就还能继续用于预测和演示。

## 7. 第六步：运行预测

训练完成后，执行：

```bash
python main.py --mode predict --config configs/base.yaml
```

这一步会：

- 加载训练好的最佳模型
- 读取 `data/test/images`
- 生成预测掩膜

结果会保存在：

```text
experiments/preds
```

## 8. 第七步：启动网页系统

运行：

```bash
python webapp.py
```

然后在浏览器中打开：

```text
http://127.0.0.1:5000
```

## 9. 网页功能怎么用

### 9.1 Single Image

用于单张图像推理。

操作方式：

1. 点击选择文件
2. 选择一张眼底图像
3. 点击 `Run Inference`

页面会显示：

- 原图
- 分割掩膜
- 叠加结果

### 9.2 Batch Inference

用于多张图像批量推理。

操作方式：

1. 多选多张图片
2. 点击 `Process Batch`

适合一次性查看多张验证图像或测试图像的分割效果。

### 9.3 Import History JSON

用于导入历史记录文件。

如果你之前保存过推理记录，可以通过这里重新加载。

## 10. 网页里该选哪里的图片

如果你想看验证效果，建议选择：

```text
data/val/images
```

如果你想做展示，建议选择：

```text
data/test/images
```

区别如下：

- `val/images` 更适合验证模型效果
- `test/images` 更适合展示最终预测结果

## 11. 当前项目支持的模型

可以在配置文件中修改模型名称：

```yaml
model:
  name: attention_aspp_unet
```

当前支持：

- `unet`
- `attention_unet`
- `residual_unet`
- `attention_aspp_unet`

配置文件位置：

`configs/base.yaml`

## 12. 当前项目支持的预处理和增强

目前已经实现：

- resize 到 512×512
- CLAHE
- 随机翻转
- 随机旋转
- 随机缩放
- 弹性变形
- 亮度和对比度增强
- 高斯噪声

相关代码在：

`src/datasets.py`

## 13. 常用命令汇总

数据准备：

```bash
python main.py --mode prepare-data --config configs/base.yaml
```

训练：

```bash
python main.py --mode train --config configs/base.yaml
```

预测：

```bash
python main.py --mode predict --config configs/base.yaml
```

启动网页：

```bash
python webapp.py
```

## 14. 常见问题

### 14.1 停止训练后模型会不会没了

不会。只要 `experiments/best.pt` 或 `experiments/last.pt` 已经生成，模型文件就还在。

### 14.2 为什么网页只能看结果，不能直接算验证指标

因为当前网页主要是演示系统，重点是推理和结果展示，不是自动评估页面。

### 14.3 为什么测试集没有真值标签

因为当前使用的 Kaggle 版 DRIVE 没有提供测试集血管真值标注，所以测试集主要用于预测展示。

## 15. 推荐使用顺序

建议按这个顺序操作：

1. 安装环境
2. 放好 DRIVE 数据集
3. 运行 `prepare-data`
4. 运行 `train`
5. 观察验证指标并选择最佳模型
6. 运行 `predict`
7. 启动 `webapp.py` 做结果展示

这样就能完整跑通整个项目流程。
