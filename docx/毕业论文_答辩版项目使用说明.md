# 基于 U-Net 的视网膜血管分割系统项目使用说明

## 1. 项目概述

本项目基于 PyTorch 深度学习框架实现视网膜血管分割任务，完成了从数据预处理、模型训练、模型推理到 Web 演示系统展示的完整流程。项目以 DRIVE 公开数据集为基础，构建了 U-Net 及其改进模型，并提供可视化分割结果输出，适合作为毕业论文实验平台与答辩展示系统。

本系统目前具备以下功能：

- 支持 DRIVE 数据集的自动整理与划分
- 支持 U-Net、Attention U-Net、Residual U-Net、Attention ASPP U-Net 等模型
- 支持 CLAHE、旋转、翻转、缩放、弹性变形等数据增强策略
- 支持 Dice、IoU、Accuracy、Sensitivity、Specificity、AUC 等指标统计
- 支持单张图像推理、批量图像推理与结果可视化展示
- 支持本地历史记录保存，便于系统演示与结果回溯

## 3. 开发环境与依赖

建议使用独立虚拟环境部署本项目，以避免与本机其他 Python 包发生版本冲突。

推荐环境如下：

- Python 3.10 及以上
- PyTorch 2.2 及以上
- Windows 10/11 或 Linux

依赖安装命令如下：

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

## 4. 数据集说明

本项目默认使用 Kaggle 版本 DRIVE 数据集。数据解压后建议存放于以下目录：

```text
data/DRIVE/
├── training/
│   ├── images/
│   ├── 1st_manual/
│   └── mask/
└── test/
    ├── images/
    └── mask/
```

说明如下：

- `training/images` 为训练图像
- `training/1st_manual` 为血管人工标注
- `training/mask` 与 `test/mask` 一般为视野区域掩膜，不直接作为血管分割真值
- 当前 Kaggle 版本不包含测试集血管真值，因此项目会自动从训练集中划分验证集

## 5. 数据准备流程

数据准备命令如下：

```bash
python main.py --mode prepare-data --config configs/base.yaml
```

该命令完成以下工作：

- 读取 `data/DRIVE/training/images`
- 读取 `data/DRIVE/training/1st_manual`
- 按配置将训练数据划分为训练集与验证集
- 读取 `data/DRIVE/test/images` 作为测试输入图像
- 自动生成 `data/train`、`data/val`、`data/test`

生成结果目录如下：

```text
data/
├── train/
│   ├── images/
│   └── masks/
├── val/
│   ├── images/
│   └── masks/
└── test/
    └── images/
```

## 6. 模型训练流程

训练命令如下：

```bash
python main.py --mode train --config configs/base.yaml
```

训练过程中系统会：

- 构建配置文件指定的分割模型
- 加载训练集与验证集
- 对输入图像执行预处理与增强
- 使用组合损失函数进行反向传播优化
- 记录每轮训练与验证指标
- 自动保存最新模型与最佳模型

训练输出包括：

- `experiments/last.pt`：最后一轮模型参数
- `experiments/best.pt`：验证集表现最优模型参数
- `experiments/metrics.json`：训练日志与指标记录

## 7. 模型预测流程

训练完成后，可通过以下命令生成测试图像的血管分割结果：

```bash
python main.py --mode predict --config configs/base.yaml
```

该命令会：

- 加载 `experiments/best.pt`
- 读取 `data/test/images`
- 执行图像预处理和模型前向推理
- 输出二值血管掩膜图

预测结果保存在：

```text
experiments/preds/
```

## 8. Web 演示系统使用方法

启动方式如下：

```bash
python webapp.py
```

浏览器访问地址：

```text
http://127.0.0.1:5000
```

系统主要功能如下：

- 单张图像上传与在线推理
- 多张图像批量推理
- 分割掩膜与叠加结果显示
- 历史结果记录与 JSON 导入

系统适用于毕业答辩中的实验结果演示与交互展示。

## 9. 常用评价指标说明

本项目当前训练阶段支持以下评价指标：

- Dice 系数：衡量预测区域与真实区域的重叠程度
- IoU：衡量预测与真实区域的交并比
- Accuracy：像素分类整体准确率
- Sensitivity：血管像素召回能力
- Specificity：背景像素识别能力
- AUC：模型对正负样本整体区分能力

其中，毕业论文中通常建议重点报告：

- Dice
- IoU
- Sensitivity
- Specificity
- AUC

## 10. 当前模型与实验配置

当前默认配置文件为 `configs/base.yaml`，其主要配置包括：

- 模型类型：`attention_aspp_unet`
- 输入通道数：`3`
- 输出通道数：`1`
- 图像尺寸：`512`
- 批次大小：`4`
- 学习率：`0.0003`
- 训练轮数：`80`
- 损失函数：`Dice Loss + Focal Loss`

用户可根据实验需求修改该配置文件，实现不同模型与训练参数的对比实验。

## 11. 建议的实验流程

为了满足毕业论文实验设计要求，建议按以下流程开展实验：

1. 使用 `prepare-data` 完成数据整理与划分
2. 使用基础 U-Net 模型训练，记录基线结果
3. 切换为注意力模型、残差模型、多尺度模型，进行改进实验
4. 比较不同模型的验证集 Dice、IoU 等指标
5. 选取最佳模型进行测试集预测与可视化展示
6. 启动 Web 系统进行结果演示与答辩展示

## 12. 常见问题说明

### 12.1 训练中途停止后模型是否丢失

不会。训练过程中会持续保存 `best.pt` 与 `last.pt`，即使中断训练，已经保存的模型文件仍可继续用于推理和演示。

### 12.2 网页中应选择哪些图像进行验证

若进行人工验证，建议选择：

```text
data/val/images
```

因为该目录对应验证集图像，更适合观察模型泛化效果。

### 12.3 为什么测试集不能直接计算定量指标

因为当前 Kaggle 版本 DRIVE 未提供测试集血管真值标注，只提供测试图像与视野掩膜，因此适合用于分割结果展示，不适合直接进行严格定量评估。
